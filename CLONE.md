# Cloning inpres-a → a new company submission (inpres-x / inpres-t)

`inpres-a` is the **master template**. A company clone changes only TWO things —
the **look** (`resources/css/theme.css`) and the **branding** (`config/brand.php` + `.env`).
Layout/structure stays identical across all clones.

---

## 1. Copy the folder

```
inpres-a  →  inpres-x   (or inpres-t)
```

## 2. Re-skin: edit ONE file — `resources/css/theme.css`

Everything that controls colour + fonts is the **COMPANY BRAND block** at the top of
`resources/css/theme.css` (between the two boxed comment banners). Change only those values:

- **Brand colours:** `--teal` (primary), `--pine` (dark/headings), `--orange` (spotlight),
  `--ink-navy` (dark text/ink on wizard + officer cards) and their scales. These feed the public
  Portal, the officer System, and the /apply wizards.
- **RGB triplets:** `--brand-rgb`, `--pine-rgb`, `--orange-rgb`, `--ink-navy-rgb` — **must match**
  the hexes above (they drive the `rgba()` glows/tints). If you change `--teal`, change
  `--brand-rgb` to match (same for the others).
- **Fonts:** `--ai-font` (portal shell), `--font-sans` (portal skin + system), and the `--*-mono`
  stacks. Also update the Google-Fonts `@import` at the very top of theme.css to load whatever
  families you set. (Self-hosted bunny fonts in `vite.config.js` are optional/secondary.)

Nothing below the "END COMPANY BRAND" banner normally needs editing — it derives from the block
above. The officer console is a dark surface; its handful of dark `--ai-*` overrides live in
`resources/css/system.css` (`:root`) and already track `--pine-900` + `--teal`.

After editing, rebuild (step 5).

## 3. Re-brand: edit `.env` (and optionally `config/brand.php`)

All company names/logos/strings come from `config/brand.php`, overridable via `.env`:

```
APP_NAME=InpresX
BRAND_NAME="Portal Short Name"
BRAND_AGENCY="New Agency Full Name"
BRAND_MINISTRY="Owning Ministry"
BRAND_LOGO=img/your-logo.png
BRAND_CREST=img/your-crest.svg
BRAND_VERSION="v0.1 · inpres-x"
```

(Address/phone have `BRAND_ADDRESS` / `BRAND_PHONE` too — see `config/brand.php`.)

## 4. Drop in assets

Put the company logo + crest into `public/img/` matching the `BRAND_LOGO` / `BRAND_CREST` paths.

## 5. Build + clear caches

```
npm install
npm run build
php artisan config:clear      # picks up new .env / config/brand.php
php artisan view:clear        # flush compiled blades (absolute paths + old strings)
```

## 6. Verify

```
php artisan serve   # + npm run dev for hot reload
```
Load the portal home and one officer page; confirm colours, fonts, logo, names, version all
reflect the new company.

---

## What is NOT auto-parameterized (review per company)

The shared **chrome** (header, footer, layout brand lockups, titles, version) is fully
config-driven. But **page-level content/copy** is still JPN-specific domain text and must be
reviewed/rewritten per company:

- `app/Http/Controllers/ContentController.php` — FAQ answers, the state **office directory**,
  and service-catalogue descriptions (all modelled on real JPN facts).
- Prose mentions of "JPN" / agency inside `resources/views/content/*.blade.php`,
  `apply.blade.php`, `track*.blade.php`, `system/login.blade.php`.

(The stylised wordmark — `partials/brand.blade.php` and the officer topbar — is now
config-driven off `BRAND_NAME` and colour-tokenised, so no hand-editing needed. All per-page
inline `<style>` brand colours across the officer + wizard views have been tokenised to
theme.css, so a theme.css swap re-skins the entire app.)

Find everything left to review:

```
grep -rniE "InPreS|Jabatan Pendaftaran|JPN|inpres-a" app/ resources/views/ --include=*.php
```

(Ignore `storage/framework/views/` — compiled cache, regenerated by `view:clear`.)
